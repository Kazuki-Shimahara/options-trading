**作成日：** 2026年3月6日
**更新日：** 2026年3月7日（エージェントチームによるブラッシュアップ）
**ステータス：** 第2稿

## 🎯 アプリ概要

日経225オプション取引を対象とした、**エントリー判断サポート＋売買記録・振り返り分析** のWebアプリ。

ブラック・ショールズモデル（Merton配当調整版）をベースにIV（インプライドボラティリティ）を分析し、エントリーの最適タイミングをLINEで通知する。また、売買履歴を記録・可視化することで「なぜ勝ったか・負けたか」を定量的に振り返れるようにする。

| 項目 | 内容 |

### ペルソナ（具体化）

> 日経225オプションを月に数回トレードする個人投資家。日中は本業があり、相場を常時監視できない。エントリータイミングの判断に自信がなく、過去のトレードの振り返りもできていない。スマートフォンからトレード直後に記録をつけたい。

### 主要ユーザーストーリー

1. **「忙しい時でもIVが低い好機を見逃したくない」** → LINE通知機能の根拠
2. **「なぜ負けたかを後で振り返りたい」** → 売買記録＋敗因タグの根拠
3. **「エントリー理由を記録して、判断力を鍛えたい」** → メモ機能の根拠
4. **「含み損益とポジション状況を一目で把握したい」** → 未決済ポジション一覧の根拠
5. **「確定申告や外部分析のためにデータを出力したい」** → CSV出力機能の根拠

## 📦 機能一覧

### 機能① エントリーシグナル通知

**目的：** IV分析に基づき「今が仕込み時」を自動で通知する

#### シグナルロジック（詳細）

**データソース：** J-Quants API（JPX公式）を第一候補。日中クローズ（15:15 JST）と夜間クローズ（翌6:00 JST）の2回取得。

**BSモデル：** Merton配当調整版（配当利回り約1.5〜2.0%）。先物価格を原資産価格として使用。IV逆算はNewton-Raphson法（フォールバック：Brent法）。

**IVシグナル閾値（2段構え）：**

| シグナル種別 | 条件 | 通知内容 |
|------------|------|---------|
| 通常買いシグナル | 現在IVが20日平均より15%以上低い | 「IV低下：買い検討タイミング」 |
| 強買いシグナル | 現在IVが60日平均より20%以上低い、かつ20日平均も下回る | 「強IVシグナル：買い好機」 |
| 売りシグナル | 現在IVが20日平均より20%以上高い | 「IV上昇：売り/ヘッジ検討」 |

**補助指標：**

| 指標 | 計算方法 | 活用方法 |
|------|---------|---------|
| HV（20日/60日） | 過去リターンの標準偏差×√252 | IV/HV比の算出 |
| IV/HV比 | 現在IV ÷ HV20 | 1.0未満=買い好機、1.5超=売り好機 |
| 日経VI | JPX公表値（J-Quants取得） | 20未満=低ボラ、30超=高ボラ |
| IVランク | (現在IV - 1年最低) / (1年最高 - 1年最低) × 100 | 0-25=緑(買い好機)、75-100=赤(売り好機) |
| IVパーセンタイル | 過去1年で現在IVより低かった日数/252×100 | IVランクの補完（外れ値に強い） |
| プット/コールレシオ(PCR) | プット出来高÷コール出来高 | 1.2超=逆張り買いシグナル補強 |

**シグナル除外フィルタ：**
- SQ前3営業日はシグナル感度を下げる（ガンマ・ベガが急変するため）
- 対象オプション：残存5営業日以上、ATMから±2000円以内（流動性確保）

**取引スタイル別フィルタ（設定画面で選択）：**
- 買い中心 → IVランク25以下のシグナルのみ通知
- 売り中心 → IVランク50以上のシグナルのみ通知
- 全表示 → 全シグナルを通知（上級者向け）

**通知手段：** LINE Messaging API（月200通まで無料）

### 機能② 売買履歴の記録

**目的：** 取引のたびに「なぜ入ったか」をメモして記録する

#### 記録項目

| 種別 | 項目 |
|------|------|
| 定量情報 | 取引日時、コール/プット、権利行使価格、限月（expiry_date）、枚数、購入価格、決済価格、決済日（exit_date）、損益 |
| 定性情報 | エントリー理由（テキストメモ）、エントリー時のIV水準、IV/HV比 |
| 市場環境タグ | リスクオン/オフ、トレンド軸、ボラ軸、イベント軸（後述） |
| 敗因タグ | 固定15タグ（複数選択可）＋自由メモ |
| Greeks（自動計算） | エントリー時のΔ・Γ・Θ・ν（BSモデルで自動計算） |

#### UX要件
- **モバイルファースト設計**（スマートフォンからトレード直後に入力するユースケース）
- コール/プット選択はボタン切り替え（ドロップダウン不使用）
- 下書き保存機能（途中で中断しても消えない）
- 取引一覧のフィルタリング（コール/プット、期間、ステータス）
- **編集・削除機能**（記録ミスは必ず起きる）
- **決済入力機能**（open → closed へのステータス更新、決済価格・日付入力）

#### 市場環境タグ分類

| 軸 | タグ |
|----|------|
| リスク | `リスクオン` / `リスクオフ` |
| トレンド | `上昇トレンド` / `下降トレンド` / `レンジ相場` |
| ボラ | `高ボラ(VI25超)` / `通常ボラ(VI18-25)` / `低ボラ(VI18未満)` |
| イベント | `イベント前(3営業日以内)` / `イベント後` / `SQ週` / `決算シーズン` / `通常` |

#### 敗因タグ固定リスト（15個）

**エントリー判断ミス：**
1. `IV高値掴み` - IV水準が高い時に買いエントリー
2. `トレンド逆張り` - 明確なトレンドに逆らったポジション
3. `イベント前不用意エントリー` - 重要イベント直前にボラ考慮せずエントリー
4. `方向性判断ミス` - 相場の方向を読み間違えた

**タイミングミス：**
5. `エントリー早すぎ` - 想定した動きの前にポジションを取りすぎた
6. `エントリー遅すぎ` - 動き出してから追いかけてプレミアムが高かった
7. `SQ接近リスク` - 残存日数が少ない状態でのポジション保持

**エグジット判断ミス：**
8. `損切り遅れ` - 含み損が拡大してから対処
9. `利確早すぎ` - 利益を伸ばせる局面で早期利確
10. `利確遅すぎ` - 利益が出ていたのに戻ってしまった

**リスク管理ミス：**
11. `ポジションサイズ過大` - 枚数が多すぎてリスク過大
12. `デルタ偏り` - ポートフォリオのデルタが偏っていた
13. `証拠金不足` - 証拠金管理が不十分で強制決済

**外部要因：**
14. `想定外イベント` - 予期せぬニュース・地政学リスク等
15. `流動性不足` - 板が薄く想定通りの価格で約定できなかった

### 機能③ 振り返り・敗因分析

**目的：** 「なぜ負けたか」をデータで可視化して改善につなげる

| 分析軸 | 内容 |
|--------|------|
| IVタイミング分析 | エントリー時のIVランク別勝率比較 |
| 損益チャート | 時系列での損益推移グラフ（Recharts） |
| 敗因タグ集計 | タグ別の負けトレード数・損失額の集計・ランキング |
| 市場環境×勝率 | 環境タグ別の勝率ヒートマップ |
| IV/HV比分析 | エントリー時のIV/HV比と損益の相関 |
| ボラティリティ・スキュー | OTMプットIV - ATM IV（Phase 3以降） |

### 機能④ ポジション管理

**目的：** 未決済ポジションの含み損益をリアルタイムで把握する

| 機能 | 内容 |
|------|------|
| 未決済ポジション一覧 | ホーム画面にサマリー表示 |
| 含み損益計算 | 現在のBSフェアバリューと購入価格の差から算出 |
| ポートフォリオGreeks | 全ポジションの合算Δ・Γ・Θ・ν |
| デルタ中立乖離度 | ポートフォリオデルタが設定閾値超えでLINE警告 |
| 最大損失表示 | 買いポジション：プレミアム全額。売りポジション：±10%変動時の損失 |

### 機能⑤ イベントカレンダー

**目的：** 重要イベントをトレード記録に紐付け、イベント影響を分析できるようにする

| SQ日自動表示 | 毎月第2金曜日（メジャーSQ：3/6/9/12月）をハイライト |

## 🛠 技術スタック

> HTML/CSSの知識があり、Claude Codeで実装する前提での構成

| 層 | 技術 | 採用理由 |
|----|------|---------|
| フロントエンド | Next.js（React） | Webアプリとして動かしやすく、情報も豊富 |
| バックエンド | Next.js API Routes + Server Actions | フロントと同じリポジトリで管理しやすい |
| データベース | Supabase | 無料枠あり・マルチユーザー対応が簡単 |
| 通知 | LINE Messaging API | 無料で月200通まで |
| 価格データ取得 | J-Quants API（JPX公式）| 国内取引所の公式データ。オプション価格取得可能 |
| グラフ | Recharts | React向け・実装が容易 |
| デプロイ | Vercel | Next.jsと相性が良く無料枠あり |

### 技術的注意事項

- **Supabaseクライアント：** サーバー用（`createServerClient`）とクライアント用（`createBrowserClient`）を`@supabase/ssr`で分離（Phase 2以降）
- **BS計算：** フロントエンドで実行（純粋な数値計算のため）。IVシグナル判定（DB参照）はAPI Routesで実装（ハイブリッド構成）
- **Vercel Cron：** Hobbyプランは1日1回のみ。頻度を上げたい場合はcron-job.org等の外部サービスからAPI Routeを叩く（無料・1分間隔可能）
- **J-Quantsトークン：** リフレッシュトークンの有効期限は1週間。Supabaseにトークンを保存し自動更新するロジックが必要
- **Server Actions：** クライアントから直接Supabaseにinsertする現状の実装は技術的負債。Phase 2までにServer Actionsへ移行
